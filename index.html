Step-By-Step

Installation Wizard

Follow these steps to flash the AI Assistant firmware and connect your Flipper Zero to the WiFi Dev Board. Click a step to expand it.

1
Requirements & Downloads

Before you start, make sure you have the following:

  • Flipper Zero device (any hardware revision)
  • WiFi Dev Board (ESP32-S2 module)
  • USB-C cable (data capable)
  • Computer running Windows 10+, macOS 12+, or Linux
  • qFlipper desktop app installed

Download the latest AI Assistant firmware release:

https://github.com/NaTo1000/flipper-zero-ai-assistant/releases/latest

Also install qFlipper from the official site:

https://flipperzero.one/update
2
Back Up Your Flipper

Always back up before flashing. Connect your Flipper via USB-C, open qFlipper, and click Backup.

Alternatively, back up from the Flipper menu:

Main Menu → Storage → Backup → SD Card

Your SD card data (signals, NFC cards, keys) will be preserved across firmware updates.

3
Flash AI Assistant Firmware

Unzip the downloaded release. In qFlipper, choose Install from file and select the .dfu package:

flipper-zero-ai-assistant-vX.X.X.tgz

Or flash via CLI using the ufbt tool:

pip install ufbt ufbt flash FIRMWARE=./flipper-zero-ai-assistant-vX.X.X.tgz

Wait for the progress bar to complete. The device will reboot automatically.

4
Flash the WiFi Dev Board

Attach the WiFi Dev Board to the GPIO header on the back of your Flipper. Hold the BOOT button on the Dev Board while connecting USB.

Flash the ESP32-S2 firmware using esptool:

pip install esptool esptool.py --chip esp32s2 --port /dev/ttyUSB0 write_flash \ 0x0000 bootloader.bin \ 0x8000 partitions.bin \ 0x10000 ai-assistant-wifi.bin

On Windows, replace /dev/ttyUSB0 with the correct COM port (e.g., COM5).

5
Configure WiFi & API Key

After flashing, the Dev Board broadcasts a setup hotspot. Connect your phone or computer to it:

SSID: FlipperAI-Setup Password: flipper0

⚠ This is a temporary setup password. Change it in the web UI after configuration.

Open a browser and navigate to:

http://192.168.4.1

Enter your home WiFi credentials and your OpenAI API key (or compatible LLM endpoint). Save and reboot the Dev Board.

⚠ Security notes: (1) Change the setup hotspot password immediately after first use. (2) Use a restricted API key with a spending limit — never share it or commit it to source control.

6
Verify & Start Using

Once connected to your home network, the orange LED on the Flipper pulses to confirm WiFi link. Open the AI Assistant app on your Flipper:

Main Menu → Apps → GPIO → AI Assistant

You can also access the web UI from any browser on your local network:

http://flipperai.local

✓ Installation complete! Your Flipper Zero AI Assistant is ready.

Help

Frequently Asked Questions

Is this legal to use?
The Flipper Zero is a legitimate security research tool sold commercially. It is legal to own and use for authorised testing on systems you own or have written permission to test. Misuse of radio signals (e.g., interfering with licensed spectrum) may violate local laws. Always act responsibly.
Do I need the WiFi Dev Board for the AI features?
Yes. The AI Assistant firmware requires the ESP32-S2 WiFi Dev Board to connect to your local network and relay requests to the configured LLM API endpoint. The base firmware still works without the Dev Board but AI features will be disabled.
Which AI models are supported?
The assistant supports any OpenAI-compatible API endpoint. This includes OpenAI GPT-4o, GPT-3.5-turbo, Anthropic Claude (via proxy), Mistral, and locally-hosted models served by Ollama or LM Studio.
Will flashing this firmware void my warranty?
Third-party firmware may affect your hardware warranty. You can always restore the official Flipper Zero firmware via qFlipper, which will return the device to its original state. Back up your SD card data before flashing.
The orange LED is not pulsing after setup — what do I do?
Check that the Dev Board is seated firmly on the GPIO pins. Verify your WiFi credentials are correct by re-entering setup mode (hold the BOOT button on the Dev Board for 3 seconds). Ensure your router uses 2.4 GHz (the ESP32-S2 does not support 5 GHz). Check the serial monitor for error messages.
Where can I get support or contribute?
Open an issue or pull request on the GitHub repository. Community discussion happens in the Issues tab. Contributions are welcome — see CONTRIBUTING.md for coding standards and the PR process.
